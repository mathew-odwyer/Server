# Winterhaven - MVP Specification Document

## Summary

Winterhaven is a 2D top-down online role-playing game showcasing expertise in distributed systems, real-time networking, secure authentication, and multiplayer game architecture. This MVP demonstrates production-ready skills in containerized microservices, database design, and real-time client-server communication.

**GOAL:** A functional multiplayer RPG with secure authentication and real-time movement.

---

## Technical Architecture

| Component | Technology | Purpose |
|-----------|-----------|---------|
| Client | GameMaker (GML) | Game rendering, input handling, client-side prediction |
| Game Server | GameMaker (GML) | Real-time game logic, state authority, world simulation |
| Web API | ASP.NET Core, C# | Authentication, user management, data persistence |
| Database | MSSQL + EF Core | Player data, accounts, persistent storage |
| Reverse Proxy | Caddy | SSL/TLS termination, routing |
| Orchestration | Docker Compose | Container management, service coordination |

## Completed Features

### Authentication & Security
- [x] User registration via game client
- [x] User login via game client
- [x] ASP.NET Identity integration
- [x] JWT-based authorization
- [x] JWT refresh token mechanism
- [x] Single-session enforcement
- [x] SSL/TLS encryption (Caddy reverse proxy)
- [x] Automatic logout on disconnect

### Infrastructure
- [x] Containerized microservices (Docker)
- [x] MSSQL database with Entity Framework Core
- [x] Real-time WebSocket communication
- [x] Health checks for all services
- [x] JSON-RPC 2.0 Protocol

### Gameplay & Networking

- [x] Client-side prediction with immediate local response
- [x] Server reconciliation with authoritative position validation
- [x] Entity interpolation for smooth multiplayer rendering
- [x] Network resilience with lag compensation and input replay
- [x] 20Hz server tick rate
